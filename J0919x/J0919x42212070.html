Python 리스트 (list)

  • Python의 리스트
    • 한 개의 값(value; object)으로써 [와 ]으로 둘러싸인 값으로 구분된다.
    • 비워 있을 수도 있으며, 한 개의 예를 들어 또 다른 리스트가 들어있을 수도 있을 뿐만 아니라 한 개 이상의 또 다른 형태의 데이터들로 구성되어 있을 수도 있다.
    • 구성 요소들은 쉼표(, comma)로 구분된다.
    • 구성 요소 중 일부를 뺄 수도 있으며, 새로운 구성 요소를 추가할 수도 있으며, 구성 요소 일부를 수정도 할 수 있으며, 등등 모든 기능이 허락된다.

  • 리스트와 사칙연산
    • 위 코드를 보면 리스트에 리스트를 더하는 것은 허락되는 사칙연산이다.
      • 합하는 두 리스트를 순서데로 합해서 본래 리스트의 구성 요소 수가 늘어난다.
    • 리스트에 정수를 곱하는 것도 허락되는 연산이다.
      • 본래 리스트를 정수만큼 복사해서 추가하여 본래 리스트의 구성 요소 수가 늘어난다.
    • 위 코드를 다시 한 번 유심히 살펴보면 합셈과 곱셈 연산을 실행한 후 아무런 반환값이 없음을 볼 수 있다. 본래 리스트 내용이 바뀌었을뿐 반환하는 값이 없다는 사실을 다시 한번 곰곰히 따져 보라.

  • dot 표기법
    • 위 코드에는 숫자만을 다룬 코드에서는 볼 수 없었던 새로운 표기법이 보이고 있다.
    • 앞서 기술한데로 Python의 리스트는 구성 요소의 변경이 가능한 object이기 때문에 이 기능에 걸맞는 내장함수 (method)가 함께 따라온다는 사실이다.
      • 사실 정수와 실수 모두 내장함수가 따라오는 object들이지만 전혀 다루지 않았다는 정도만 알고 넘어가자.
    • 위 코드 첫줄은 이름이 a인 변수에 3 개의 데이터들로 구성된 리스트를 연결하였다.
      • 위 표현은 바로 리스트 object 한 개를 생성했다는 뜻이다.
      • 생성된 리스트 object의 보다 정확한 표현은 리스트 object의 instance를 한 개 생성했다는 뜻이다.
    • 리스트 object의 instance 생성에 관해서는 추후 class 관련되 참고문헌을 참고하라. 여기서는 몰라도 된다.
    • 리스트 object에 이미 추가로 제공된 다양한 method들이 있으며 이들을 새로 막 만든 변수에 연결해서 사용이 가능하다.
    • 사용 가능한 method들의 목록을 보려면 Python built-in function dir()를 이용한다.

  • 리스트 내장함수 목록
      • method 설명 반환 유/무
      append(obj) 리스트 뒤에 주어진 값(obj) 추가 무
      clear() 리스트 구성요소 모두 삭제 무
      copy() 본래 리스트의 shallow 복사본 반환 유
      count(value) 리스트 구성요소들 중 value의 횟수 유
      extend(iterable) 값 여러 개를 리스트에 추가 무
      index(value) 리스트 구성 요소들 중 value의 첫번째 순서번호 반환 유
      insert(index, obj) 주어진 순서번호(index)에 obj 삽입 무
      pop(index=-1) 주어진 순서번호(index, 맨 마지막 기본)에 위치한 구성요소 제거하고 반환 유
      remove(value) 구성요소들 중 첫번째 value 제거 무
      reverse() 구성요소 순서를 뒤집는다 무
      sort(*, reverse=True) 구성요소의 오름차순 정렬 (또는 내림차순 정렬) 무
    • 위 테이블에서 보듯이 내장함수를 사용할 때는 반드시 괄호 (와 )를 사용한다 (이유는 내장함수는 함수들이기 때문).
    • 내장함수에 따라 값을 반환하지 않는 경우, 실제로 None을 반환한다.
    • 위 내장함수를 아는만큼 리스트를 멋지게 활용할 수 있다.
      • 하지만 외우려고 고생하지 말고 시간이 흘러 손가락이 익히도록 연습을 많이 하기를 권한다.

  • indexing
    • 앞에서 "순서번호"와 "index" 용어를 여러 군데 사용하였다. 두 용어는 동일하다.
    • 두 용어는 리스트 사용할 때 가장 기초적인 내용으로 리스트 구성요소의 순서를 가리킨다.
    • 구성요소 순서 표기에는 positive indexing과 negative indexing의 두 가지 방법이 있다
      • positive indexing: 맨 앞의 구성요소를 0-번째 그 다음 구성 요소는 1-번째 등으로 표기하는 순서
      • negative indexing: 맨 뒤의 구성요소를 (-1)-번째, 뒤에서 두 번째 구성요소는 (-2)-번째로 표기하는 순서
      • positive indexing으로 맨 뒤의 구성요소는 리스트 구성요소의 갯수보다 항상 1만큼 작다.
      • negative indexing으로 맨 앞의 구성요소는 리스트 구성요소의 갯수에 음수를 곱한 값이다.
    • 리스트 구성요소의 순서는 항상 두 가지 방법으로 표기 가능하며, 필요에 따라 적절히 섞어서 사용할 수 있는 편리함이 있다.

      • 시작번호 끝번호 사이에 들어있는 모든 구성요소를 뜻한다.
      • 시작번호를 생략하는 경우는 맨 앞부터 시작하는 것을 뜻하며,
      • 끝번호가 생략된 경우는 맨 끝까지를 뜻한다.
      • 만약 시작번호와 끝번호 모두 생략되었다면, 위 코드 맨 마지막 경우와 같이, 당연히 처음부터 끝까지, 구성요소 전부를 뜻한다.
    • 만약 시작번호와 끝번호와 함께 그 사이 모든 구성요소 대신 일부분만을 다루고 싶은 경우:

      • 리스트의 내장함수 sort()을 사용하면 반환 값이 없는 대신 해당 리스트가 바뀌며,
      • built-in function sorted을 사용하면 정렬된 리스트가 반환된 반면 본래 리스트는 변함이 없다.
