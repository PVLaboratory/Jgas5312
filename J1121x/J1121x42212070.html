Comprehension과 여러 가지 응용

  • List comprehension
    • List comprehension은 반복문을 통해 리스트 값을 만드는 과정을 한 줄로 줄이는 기법이다.
    • 단순한 리스트 만들기
      • x = [f for f in range(5, 15)]
      x = list(range(5, 15))
    • 조건을 만족하는 값들로 구성되는 리스트 만들기
      • x = [f for f in range(5, 15) if not f%3]
    • 조건에 따라 달라지는 값들로 구성되는 리스트 만들기
      • x = [f if f%4 == 1 else f**2 for f in range(5, 25)]
  • List comprehension과 built-in functions
    • List comprehension 기법으로 만든 리스트는 여느 리스트처럼 built-in function()에 바로 사용할 수도 있다.
    • List comprehension과 sum(), min(), max(), sorted()
      • x = sum([f if f%4 == 1 else f**2 for f in range(5, 25)])
      x = sorted([f for f in range(5, 15) if not f%3])
    • List comprehension과 enumerate()
      • colors=['white', 'red', 'black', 'white', 'green', 'blue']
      
indices=[j for j, color in enumerate(colors) if color == 'white']
    • List comprehension과 zip()
      • colors=['white', 'red', 'black', 'white', 'green', 'blue']
      
indices=[5, 9, 0, 6, 7, 9]
      
indx2find=9
      
answers=[c for c, j in zip(colors, indices) if j == indx2find]
    • List comprehension과 str
      • colors=['white', 'red', 'black', 'white', 'green', 'blue']
      
longString = ''.join(sorted(colors))
  • List comprehension과 list methods
    • 마찬가지로 리스트의 다양한 method()를 바로 적용할 수도 있다.
  • Dictionary comprehension
    • 그 뿐만 아니라 dict 개체도 만들 수 있으며,. 
    colors=['white', 'red', 'black', 'white', 'green', 'blue']
    
indices=[5, 9, 0, 6, 7, 9]
    
indx2find=9
    
answers={c:j for c, j in zip(colors, indices) if j == indx2find}
    List comprehension과 모양은 매우 비슷하나 리스트 생성 대신 generator를 만드는 방법도 있다. 방법은 list comprehension도 동일하나 단지 양쪽 끝에 사용하는 대괄호 대신 보통 괄호를 사용하는 차이점이 있다. 보다 중요한 차이는 generator는 해당 코드가 실행할 때 내용을 모두 실행하는 것이 아니라 실제 실행을 요청하는 코드에 따라 한 번씩 실행하게 된다. 결과적으로 메모리를 거의 사용하지 않는다는 특징 및 장점을 갖고 있다. 
    x = (f**2 for f in range(5, 15))
    원하면 또 다른 list comprehension과 중첩해서 더 다양한 리스트를 만들 수도 있다. 
    matrix = [[1, 2], [3,4], [5,6], [7,8]]
    
transpose = [[row[i] for row in matrix] for i in range(2)]
    List comprehension은 분명 리스트를 구성하는 과정에서 발생하는 시간 소모를 줄일 수 있다는 장점이 있으며, 코드가 간략해진만큼 오류 가능성도 줄어드는 것도 장점도 있고, 코드가 읽기 쉬워지는 장점도 있다.
    장점이 있는 만큼 단점도 존재한다. 코드가 줄어드는 만큼 오류 찾기가 어려워진다는 사실을 알고 있어야 한다. 따라서 읽기 쉬워지는 만큼 과하지 않을 정도로 사용할 필요가 있다.
