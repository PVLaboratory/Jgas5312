  • Python의 기본 객체는 그 자체가 값(value)을 의미한다.
  • 값이란 데이터 한 개를 뜻하며, 그 한 개가 여러 개를 묶은 것을 뜻할 수도 있다.
    • 위 오른쪽 테이블에 나열된 객체들이 여러 개를 묶은 한 개 값을 뜻한다.
  • 생성된 값은 임의로 지울 수 없다.
  • 전혀 사용하지 않는 값은, Python의 판단 하에, Garbage Collection이라는 과정을 통해 삭제된다.

  • 정수 (integers)
    • Python은 숫자에 소숫점의 사용 여부에 따라 정수와 실수를 구분한다.
      • 소숫점이 있으면 무조건 실수 (floating point numbers)로 처리하며, 없으면 정수로 처리한다.
    • 실수에 대해서는 항상 8 바이트(byte) 또는 64 비트(bit)만을 사용하는 반면, 정수는 컴퓨터의 기능을 필요한 만큼 제한없이 사용한다.
      • 제한없이 사용한다는 표현에 대한 한 가지 예시는 다음과 같다:
        • >>>
        

        
>>>
        

        print(9223372036854775807+1)
        
9223372036854775808
        
print(9223372036854775807**2)
        
85070591730234615847396907784232501249
        • 64 비트(bit)으로 표현할 수 있는 최대 정수 값은 9223372036854775807 이다. 최대 정수값이란 더 이상 큰 수를 표현할 수 없다는 뜻이다.
        • 만약 최대 정수 값에 1을 더하거나 제곱하면 어떻게 될까?
        • Python은 아무런 오류없이 최대 정수값에 1을 더하면 9223372036854775808을 보여주고, 9223372036854775807을 제곱한 값으로 85070591730234615847396907784232501249을 보여준다.
      • Python은 정수를 연산하고 표현하는 데 있어서 8 바이트라는 제한을 사용하지 않는다.

  • 실수 (floating point numbers)
    • 소수점 위치가 숫자에 따라 달라지는 수
    • Python은 실수에 사용하는 64 비트 중 11 비트를 10의 지수에 사용하고 나머지 53 비트에 1보다 작은 숫자를 표현하는 데 사용한다. 당연히 지수에 사용한 11 비트 중 1 비트는 지수의 부호에 사용하며, 53 비트 중 1 비트는 숫자의 부호를 표현하는 데 사용한다.
    • Python으로 실수 사용에 있어서 반드시 알아야 두 가지 사항
      • 그동안 학교 수업을 통해 배운 실수에 대한 덧셈과 뺄셈 결과에 대해 동일한 계산을 Python으로 처리하는 경우 결과가 다를 수 있음에 유의해야 한다.
      • 아래 첫번째 코드는 0.1에 0.1을 더한 결과가 0.2와 동일한 지 확인하는 코드이며, 두번째는 0.1을 세번 더한 값이 0.3과 동일한 지 확인하는 코드이다.
        • >>>
        

        
>>>
        

        print(0.1 + 0.1 == 0.2)
        
True
        
print(0.1 + 0.1 + 0.1 == 0.3)
        
False
        • 첫번째 계산 결과는 당연하다고 이해하는 반면 두번째 계산 결과에 대해서는 쉽게 수긍하기 어려울 것이다.
        • 혹시 Python에 잘못이 있다고 생각하는 사람은 이 결과가 Python만의 문제가 아니라 모든 컴퓨터 프로그램 언어가 가진 문제점이라는 사실을 가능한 빨리 받아주어야 한다.
        • 앞서 언급한 바와 같이 64 비트만을 이용하여 실수를 표현하는 모든 프로그램 언어가 가진 문제점이며, 수년 후 64 비트 대신 128 비트 또는 512 비트를 이용하여 실수를 표현하더라도 항상 발생하는 문제점이라는 점을 빨리 깨달아야 한다.

      • epsilon: 1과 1보다 큰 가장 작은 수의 차이
        • 64 비트로 표현하는 실수는 10-308의 매우 작은 수도 표현할 수 있다.
        • 하지만 52 비트로 소숫점 이하의 값을 표현하다보니 2-52 = 2.22×10-16가 최소 분해능이 되어, 2.22×10-16의 반 보다 작은 수를 1에 더한 경우 1과 차이를 만들지 못하는 일이 발생하며, 2.22×10-16의 반 보다 큰 수를 1에 더하면 결과는 2.22×10-16를 1에 더한 값과 동일해진다.
          • >>>

          • 첫번째 계산 결과는 작은 숫자와 더한 결과 분명히 1과 다름을 보여주고 있는 반면, 두 번째 계산 결과는 1과 다르지 않음을 보여주고 있다.
      • 정수와 실수가 섞여있는 경우, 정수는 모두 실수로 처리된다.
      • 두 정수의 나눗셈도 실수로 처리된다.

    Python if 선택문

  • 선택문
    • 프로그램 언어의 가장 기본적인 구문 중 한 가지
    • 조건에 따라 실행할 할당문(assignement statements)을 결정하는 구문
  • 선택문을 구성하는 2 가지 정보
    1. 조건문
    2. 조건에 따라 실행하는 구문들
  • Python의 선택문은 오직 한 가지(if)뿐이다.
    • Python의 선택문은 switch 구문이 없다.

    • 조건 표현식
      • 프로그래밍 언어 입문자가 선택문 사용에 가장 어려움을 느끼는 부분은 당연히 선택문의 조건에 대한 적절한 표현식이다.
        • 선택문의 조건에 대한 적절한 표현식은 사실 입문자뿐만 아니라 전문가도 원숭이가 나무에서 떨어지는 확률보다 더 많이 오류를 만들기도 할뿐만 아니라 보이지 않는 오류을 만들어 며칠 밤낮을 새워야 하는 경우도 발생하는 내용이다.
        • 위 내용은 결국에는 시간이 흘러 다 알고 있다고 자부하는 사람도 오류를 만들기도 하니 처음부터 이해안된다고 걱정할 일이 하나도 없다는 내용이다.
      • 프로그램 언어는 모두 동일하게 조건의 값으로 참(true)과 거짓(false) 2 가지 논리값(Boolean value)을 사용한다.
        • Python에서는 2 개의 논리값 True와 False의 핵심단어로 고정적으로 사용한다 (대소문자 반드시 구별).
      • Pythonic한 조건 표현식의 한 가지 예:
        • 다른 프로그램 언어에서는 별 이의없이 사용하는 방식으로 우리는 가끔 "참이 참이냐?"라는 형태의 조건표현식을 사용하곤 한다.
        • Python에서는 동일한 내용을 "참이냐?"라는 간단한 조건표현식이 가능하다.
        • 조건에 대한 표현식 if n%2 == True:이 바로 "참이 참이냐?"라는 코드에 해당하며 표현식 if n%2:이 바로 "참이냐?"라는 코드에 해당한다.
          • 표현식 if n%2:은 이름이 n인 변수에 연결된 값을 2로 나눈 나머지가 1이면을 뜻한다.
          • Python은 0을 False처럼 처리하고 그 나머지는 모두 True으로 처리해준다.
    • 관계 연산자
      • 연산자 코드 설명
      == a == b 변수 a에 연결된 값이 변수 b에 연결된 값과 동일하면 참
      != a != b 변수 a에 연결된 값이 변수 b에 연결된 값과 다르면 참
      > a > b 변수 a에 연결된 값이 변수 b에 연결된 값보다 크면 참
      >= a >= b 변수 a에 연결된 값이 변수 b에 연결된 값보다 크거나 같으면 참
      < a < b 변수 a에 연결된 값이 변수 b에 연결된 값보다 작으면 참
      <= a <= b 변수 a에 연결된 값이 변수 b에 연결된 값보다 작거나 같으면 참
      • 관계 연산자의 2 개의 기호는 반드시 함께 사용해야 하며
      • 부등호와 같이 사용된 등호는 반드시 뒤에 적어야 한다.

    • 논리 연산자
      • 연산자 코드 설명
      and cond1 and cond2 cond1도 참이고 cond2도 참인 경우에만 참 그외는 모두 거짓
      or cond1 or cond2 cond1과 cond2 둘 중 하나라도 참이면 참
      not not cond cond이 거짓이면 참, 참이면 거짓
      • 표현식 cond1 and cond2에서 만약 cond1이 거짓으로 판정하면 cond2이 참인지 거짓인지 판별하는 과정도 생략한다 (이유는 각자 생각).
      • 마찬가지로, 표현식 cond1 or cond2에서 만약 cond1이 참으로 판정하면 cond2이 참인지 거짓인지 판별하는 과정도 생략한다 (이유는 각자 생각).

    Python 반복문

    • 반복문은 말 그대로 구문(들)을 반복해서 실행하라는 정해진 규칙으로 구체적으로 2 가지 정보와 함께 반복문 사용에 필요한 정보 한 가지를 알아야 한다.
      • 2 가지 정보
        1. 반복횟수
        2. 반복하는 구문(들)
      • 반복횟수에 따라 반복해주는 방법 -- 프로그램은 어떻게 알고 처리할까?
        • counter 변수
    • Python의 반복문은 기본적으로 리스트에 들어있는 항목들을 순서데로 한 개씩 갖고 와서 반복문 구문들를 실행한다.
      • for item in ["apple", "banana", 5]:
      
  if isinstance(item, int):
      
    print(item**10)
      
  else:
      
    print(item)
      # 결과물
      
apple
      
banana
      
9765625
      • 위 코드에서 이름이 item인 변수를 counter라 부르며, 매번 리스트에 갖고 온 항목마다 연결된다 ("연결된다"의 의미를 모른다면 질문하세요).
      • 위 예제와 달리 만약 원한다면 리스트에서 갖고온 항목을 사용하지 않아도 된다. 사용하지 않을 항목을 굳이 갖고 올 필요도 없다면, 처음부터 리스트를 사용할 필요도 없을 것이다.
    • 단지, 정해진 반복 횟수만큼 특정 구문을 반복하고 싶을 때 필요한 리스트를 만들어주는 python built-in 함수가 바로 range()이다. 예를 들어, 단순히 10이라는 숫자를 3 번 출력하고 싶으면 아래 코드처럼 counter variable 대신 밑줄(underscore, _)를 활용한다.

  print(10)

    • Python의 built-in function range()의 사용법에 관한 설명으로 한 가지 예를 생각해 보자. 정수 53부터 125까지 홀수들의 합을 구한다고 해 보자. 아래 코드는 for 반복문을 이용한 방법의 코드이다.

    total += j

    • 위 코드와 같이 Python의 built-in function range()의 괄호 안에 적는 내용이 앞에서 처럼 한 개의 정수일 수도 있으며, 이번처럼 세 개의 정수이기도 하다. 원하는 경우 두 개의 정수를 적기도 한다.

      • 정수 2 개가 있으면, 첫 번째 정수부터 두 번째 정수 미만의 자연수들을 나타내며,
      • 정수 3 개가 있으면, 첫 번째 정수부터 두 번째 정수 미만까지 세 번째 정수 간격의 자연수들을 나타낸다.
    • 위 홀수들의 합을 구하는 코드보다 더 좋은 코드가 있다고 생각이 들면 당신은 코딩 전문가이다. 아직 모르면 이제 초보자 입장에서 좀 더 배워야 할 것이 있음을 알아두는 것도 좋은 지침이다.
