Python과 functions

  • 함수를 사용하는 이유
    • 반복성: 설명 불 필요
    • 추상화: 스마트폰의 앱을 누군가가 잘 만들어 놓으면 사용자는 앱 내부에서 어떤 일을 어떻게 하든지 알 필요없이 필요한 결과를 손 쉽게 얻게 만드는 과정
      • 추상화에 대한 위의 편리한 장점에 비해 그런 장점을 얻기 위한 프로그래밍 과정의 어려움을 알고 있어야 하겠다.
    • 함수 사용에 있어서 필요한 4 가지 정보:
      1. 함수이름 (function name)
      2. 함수가 하는 일 (activity that function does)
      3. 입력인자 (input argument)
      4. 반환 값 (return valu)
  • 함수의 반환 값
    • 반환 값이란 용어 그대로 함수가 계산해서 최종적으로 내보내는 값으로써 함수를 호출하는 입장에서는 가장 중요한 정보이다.
    • Python 함수의 반환 값은 항상 1 개다.
    • 만약 여러 개의 정보를 반환 받고 싶다면 어떻게 할까?
      • 함수가 여러 값을 만들어 내 보내면 Python이 알아서 tuple로 묶어서 한 개의 tuple 값을 내 보낸다.
    • 만약 반환 값이 없는 경우, 아무 것도 내 보내지 않아도 되는지?
      • 2 가지 가능성.
        1. 아무 것도 반환하지 않는다.
        2. return None이라고 적는다.
      • 실제로는 None을 항상 내 보낸다.

    def roots4QuadEq(a):
    
  D = a[1]**2 - 4*a[0]*a[2]
    
  if not D:
    
    return -a[1]/2/a[0]
    
  elif D>0:
    
    return ((-a[1]+D**0.5)/2/a[0], (-a[1]-D**0.5)/2/a[0])
    
  else:
    
    return (complex(-a[1]/2/a[0], (-D)**0.5/2/a[0]), complex(-a[1]/2/a[0], -(-D)**0.5/2/a[0]))

              • 만약 본인은 여기 적힌데로 모두 잘 적었는 데, python에 심각한 오류 때문에 생간 문제라고 생각하는 사람이 있다면 가능한 빨리 그 생각을 머리 속에서 지워야 프로그래밍에 보다 빨리 접근할 수 있다고 말할 수 있다.
            • 해당 함수 호출 코드의 입력인자를 잘 보면 함수에서 본래 사용하는 변수 a에는 3을 입력하고 다른 변수에는 5를 입력하라고 함수호출을 했으나,
            • Python의 함수에 따르면, 입력된 첫번째 값 5를 함수 선언에 기록된 자료에 따라 변수 a에 입력된 5를 연결시킨다. 그런데, 그 다음 입력인자는 a =3이라 했으니, 해당 변수 a에 또 다른 값을 3을 연결하려고 하려는데, 보니 이미 연결된 5가 있음을 알게 된다. 결과적으로 Python입장에서는 2 개의 값이 들어왔는 데 어느 것을 연결할 지 혼란스러운 일이 발생한 것이다. 모든 프로그램 언어가 제일 싫어하는 것은 혼란스러운 일이다 (사람처럼). 모든 프로그램 언어는 혼란스러운 일이 발생하면 가장 쉽게 처리한다. 오류메세지를 내보내고 손을 턴다.
          • 위 상황은 오직 Python에서만 일어나는 일이다. 그래서 본 절의 제목을 Pythonic coding이라 적은 것이다.
      • Pythonic coding #2 (Namespaces)
        • 함수를 사용하는 단계는 현재 작성하고 있는 프로그램의 줄 수가 어느 정도 되며 그에 따라 사용하는 변수의 갯수 또한 한 손으로는 부족할 정도일 것이다.
        • 어느 순간 함수 때문에 발생하는 오류때문에 머리털을 쥐어뜯는다면 벌써 파이썬의 입문자 입장은 벗어난 상태이니 계속 아래 내용을 살펴보기 바란다.
        • 아래 도표는 Python의 namespaces, scope 그리고 name binding에 관한 자료이다 (출처: https://realpython.com/python-namespaces-scope/)
        • Python은 변수 목록을 관리하는 namespace을 위 그림과 같이 4 단계로 구분한다.
          1. Built-in namespace
          2. Global namespace
          3. Enclosing namespace
          4. Local namespace

        • 상세한 설명은 나중으로 미루고, 여기서는 glabal namespace와 local namespace만 살펴보고자 한다.
        • 용어 사용에 있어 조금 혼란스러운 점은 global이라고 해서 진짜 global이 아니며, local 이라고 해서 실제 local이 아니다.
        • 파일을 열고 시작하는, 보통 프로그램 언어에서는 main이라 부르는, 코딩 영역에서 사용하는 모든 변수들은 global namespace에서 볼수 있으며, 함수 안에서 사용하는 변수들은 local namespace에서 관리한다.
          • 함수가 여러 개면 그 수만큼 local namespace가 생성되는 것이며,
          • 함수 안에 또 다른 함수가 있다면 그 내부 함수에도 각자 따라오는 local namespace가 생성되는 것이다.

        • 왼쪽 두 코드는 오류없이 출력결과가 나왔지만 내용이 비슷한 오른쪽 코드는 오류가 발생하였다.
          • 왼쪽 코드는 함수 안에서 global 변수에 접근이 가능하다는 사실을 보여주고 있다.
          • 중간 코드는 이름이 동일한 변수가 local에도 있으며 global에도 있을 때, global 변수는 건드리지 않는다는 사실을 보여주고 있다.
          • 그렇다면 오른쪽 코드는 왜 오류가 발생하였을까?
          • 오류 내용은 아직 알 수 없는 변수 x에 값을 연결할 수 없다는 내용이다.
            • 이 내용을 제대로 이해하려면 오른쪽 코드의 코드 작성자의 의도를 먼저 짚고 넘어가야 할 것 같다.
            • 만약 코드 작성자가 함수 안에서 local 변수 x의 값에 3을 추가하려고 했다면, 이는 코드 작성자의 잘못으로 Python이 제대로 오류를 짚은 것이다.
            • 만약 코드 작성자가 함수 안에서 global 변수 x의 값에 3을 추가하려고 했다면 (아마도 이 가정을 ...), 바로 이 부분에 대해 Python이 허락하지 않는다는 사실을 숙지할 필요가 있다.
          • 정리하면 다음 두 가지 사실을 얻을 수 있다.
            1. global 변수 이름과 동일한 이름을 local에서 가질 수 있다.
            2. local에서 global변수에 접근은 가능하지만 수정을 불가능하다.
          • 아래 코드는 위 오른쪽 코드의 오류를 피하는 방법으로 Python keyword global을 사용한 것이다.
            • 해당 키워드는 함수 안에서 이름이 x인 변수는 global namespace에 있는 것을 사용하라는 뜻이다.

    • 익명함수 (Anonymous function): lambda
      • 제목처럼 이름 없는 함수
        • 이름이 없다는 말은 호출이 어려워 굳이 따로 함수를 선언하고 호출하는 과정을 거치기 보다는 바로 사용할 수 있는 함수를 만들겠다는 의미가 강하다.
          • 호출이 전혀 불가능한 것은 아니다.
      • 입력인자에 괄호도 사용하지 않는다.
        • 괄호를 사용하면 SyntaxError 발생.
      • 반환값이라는 의미의 return도 사용하지 않는다.
        • >>>
        

        

        
>>>
        def myDivMod(a, b):
        
  return a//b, a%b
        

        
print(myDivMod(3, 5))
        
(0, 3)
        >>>
        
>>>
        x = lambda a, b: (a//b, a%b)
        
x(3, 5)
        
(0, 3)
      • 위 왼쪽 코드는 앞에서 살펴본 평범한 함수이며, 오른쪽 코드는 동일한 기능을 익명 함수로 만든 것이다.
        • 익명 함수는 그냥 바로 사용할 수도 있지만 오른쪽 코드처럼 익명 함수 object를 변수에 연결하여 사용할 수도 있다.
        • 다음으로 넘어가기에 앞서 이 글을 작성하면서 알게 된 사실을 공유하고자 한다
          • 위 두 코드를 비교하면 한 가지 의문스러운 사실을 느끼게 된다. 비록 익명 함수는 keyword return을 사용하지는 않지만 반환값만은 동일한 줄 알았는 데, 유심히 살펴보면 일반적인 함수의 여러 개의 반환값에는 따로 괄호를 사용하지 않더라고 결과적으로 항상 tuple로 묶어서 반환한다. 하지만 익명함수에서는 반드시 괄호를 사용해서 하나로 묶어서 반환해야 한다. 만약 괄호를 사용하지 않으면 NameError의 오류가 발생한다.
      • 앞서서 dictionary 형태의 값 구성요소들의 정렬에서 살펴본 내용을 다시 한 번 음미해보자.

>>>
        d = {'bananas': 2, 'apple': 4, 'pear': 1, 'orange': 3}
        
# key 값 기준 내림차순 정렬
        
sorted(d.items(), key = lambda t: t[0], reverse = True)
        
[('pear', 1), ('orange', 3), ('bananas', 2), ('apple', 4)]
        
# value 값 기준 오름차순 정렬
        
sorted(d.items(), key = lambda t: t[1])
        
[('pear', 1), ('bananas', 2), ('orange', 3), ('apple', 4)]
        
# 문자열 key 값의 글자수 기준 오름차순 정렬
        
sorted(d.items(), key = lambda t: len(t[0]))
        
[('pear', 1), ('apple', 4), ('orange', 3), ('bananas', 2)]
        • 위 예시에서는 바로 사용하여 결과를 built-in function sorted()의 입력인자 중 keyword key에 입력값으로 사용하였다.
        • 위 예시에 사용된 익명 함수의 입력인자로 사용된 t는 dictionary object의 각 구성요소를 뜻한다.
          • 각 구성요소는 key값과 value값 두 개가 한쌍으로 구성되어 있어 t[0]은 key값을 뜻하며 t[1]은 value 값을 뜻해, 각 경우에 적절하게 사용된 것이다.
