  • 연산순위
    • 연산은 기본적으로 왼쪽에서 시작해서 오른쪽으로 한 번에 한 개씩 연산한다.
    • 그렇다면, 아래 코드 계산 결과는 어떻게 처리되었을까?
      • >>>
      

      print(1+2-3*4/5%6//7**8)
      
3.0
      1. 위 계산은 가장 먼저 7**8을 계산하여 5764801을 구한다.
      2. 두 번째로 3*4/5%6//5764801을 왼쪽부터 한 번에 한 개씩 연산해서 0.0을 구한다.
      3. 그 다음, 1+2-0.0을 왼쪽부터 한 번에 한 개씩 연산해서 3.0을 구한다.
    • 괄호 사용
      • 괄호 안에 연산은 위 기본 순서에 앞서 가장 먼저 연산한다.
      • 괄호들이 중첩(nested)된 경우, 가장 안 쪽의 괄호를 가장 먼저 연산한다.

  • 복합대입 연산자
    • += x += 5 x = x + 5
    -= x -= 5 x = x - 5
    *= x *= 5 x = x * 5
    /= x /= 5 x = x / 5
    • 복합대입 연산자의 연산기호 2 개는 반드시 붙여 써야 한다.

  • 정수 (integers)
    • Python은 숫자에 소숫점의 사용 여부에 따라 정수와 실수를 구분한다.
      • 소숫점이 있으면 무조건 실수 (floating point numbers)로 처리하며, 없으면 정수로 처리한다.
    • 실수에 대해서는 항상 8 바이트(byte) 또는 64 비트(bit)만을 사용하는 반면, 정수는 컴퓨터의 기능을 필요한 만큼 제한없이 사용한다.
      • 제한없이 사용한다는 표현에 대한 한 가지 예시는 다음과 같다:
        • >>>
        

        
>>>
        

        print(9223372036854775807+1)
        
9223372036854775808
        
print(9223372036854775807**2)
        
85070591730234615847396907784232501249
        • 64 비트(bit)으로 표현할 수 있는 최대 정수 값은 9223372036854775807 이다. 최대 정수값이란 더 이상 큰 수를 표현할 수 없다는 뜻이다.
        • 만약 최대 정수 값에 1을 더하거나 제곱하면 어떻게 될까?
        • Python은 아무런 오류없이 최대 정수값에 1을 더하면 9223372036854775808을 보여주고, 9223372036854775807을 제곱한 값으로 85070591730234615847396907784232501249을 보여준다.
      • Python은 정수를 연산하고 표현하는 데 있어서 8 바이트라는 제한을 사용하지 않는다.

  • 실수 (floating point numbers)
    • 소수점 위치가 숫자에 따라 달라지는 수
    • Python은 실수에 사용하는 64 비트 중 11 비트를 10의 지수에 사용하고 나머지 53 비트에 1보다 작은 숫자를 표현하는 데 사용한다. 당연히 지수에 사용한 11 비트 중 1 비트는 지수의 부호에 사용하며, 53 비트 중 1 비트는 숫자의 부호를 표현하는 데 사용한다.
    • Python으로 실수 사용에 있어서 반드시 알아야 두 가지 사항
      • 그동안 학교 수업을 통해 배운 실수에 대한 덧셈과 뺄셈 결과에 대해 동일한 계산을 Python으로 처리하는 경우 결과가 다를 수 있음에 유의해야 한다.
      • 아래 첫번째 코드는 0.1에 0.1을 더한 결과가 0.2와 동일한 지 확인하는 코드이며, 두번째는 0.1을 세번 더한 값이 0.3과 동일한 지 확인하는 코드이다.
        • >>>
        

        
>>>
        

        print(0.1 + 0.1 == 0.2)
        
True
        
print(0.1 + 0.1 + 0.1 == 0.3)
        
False
        • 첫번째 계산 결과는 당연하다고 이해하는 반면 두번째 계산 결과에 대해서는 쉽게 수긍하기 어려울 것이다.
        • 혹시 Python에 잘못이 있다고 생각하는 사람은 이 결과가 Python만의 문제가 아니라 모든 컴퓨터 프로그램 언어가 가진 문제점이라는 사실을 가능한 빨리 받아주어야 한다.
        • 앞서 언급한 바와 같이 64 비트만을 이용하여 실수를 표현하는 모든 프로그램 언어가 가진 문제점이며, 수년 후 64 비트 대신 128 비트 또는 512 비트를 이용하여 실수를 표현하더라도 항상 발생하는 문제점이라는 점을 빨리 깨달아야 한다.

      • epsilon: 1과 1보다 큰 가장 작은 수의 차이
        • 64 비트로 표현하는 실수는 10-308의 매우 작은 수도 표현할 수 있다.
        • 하지만 52 비트로 소숫점 이하의 값을 표현하다보니 2-52 = 2.22×10-16가 최소 분해능이 되어, 2.22×10-16의 반 보다 작은 수를 1에 더한 경우 1과 차이를 만들지 못하는 일이 발생하며, 2.22×10-16의 반 보다 큰 수를 1에 더하면 결과는 2.22×10-16를 1에 더한 값과 동일해진다.
          • >>>
          

          
>>>
          

          
>>>
          print(1.12e-16 + 1 == 1)
          
False
          
print(1.11e-16 + 1 == 1)
          
True
          
print(1.11e-16 + 1 == 2.22e-16 + 1)
          
True
          • 첫번째 계산 결과는 작은 숫자와 더한 결과 분명히 1과 다름을 보여주고 있는 반면, 두 번째 계산 결과는 1과 다르지 않음을 보여주고 있다.
      • 정수와 실수가 섞여있는 경우, 정수는 모두 실수로 처리된다.
      • 두 정수의 나눗셈도 실수로 처리된다.

    Python 변수 이름(names)

    • Python 언어에서 변수란 단순히 이름뿐이다.
    • 변수는 단순히 이름으로 값과 연결되어 있을 뿐이다.
    • Python의 변수 이름(name)은 수많은 종류의 데이터를 다루는 가장 기본적인 수단이다.
      • 데이터의 종류에 상관없이 원하는 데이터를 갖고와서 처리해서 내가 원하는 조건에 부합하는 지 따져 보기 위해 제일 먼저 할 일은 바로 원하는 데이터를 원하는 이름의 변수에 연결해 주는 일이다.
      • 즉, 변수에 연결되지 않은 데이터란 상상할 수도 없다.
    • 결과적으로 변수를 보면 연결된 데이터를 상상할 수 있도록 정하는 것이 여러모로 편할 것이다.
      • Python은 의외로 변수 이름에 매우 자유스럽다. 한글도 사용할 수 있으며 그리스 기호를 사용할 수도 있다.
    • 자유롭게 이름을 정할 수도 있지만 남용할 수는 없다. 이유는 python code내에서 변수를 쉽게 구별할 수 있어야 한다.
    • 변수 이름은 가급적 영문으로 시작한다.
      • 변수 이름 내부에 숫자를 사용할 수 있다.
    • 변수 이름은 반드시 한 개의 단어이어야 한다.
      • 변수 이름에 빈 칸을 사용하고 싶다면 빈 칸 대신 밑줄(_; underscore)를 사용한다.
    • 변수의 대/소문자는 구별된다.
      • abc, Abc, aBc, abC 모두 다른 변수이름들이다.
    • Python의 keyword는 변수이름으로 사용할 수 없다.
    • Python의 built-in functions은 변수이름으로 사용할 수 있으나, 사용하면 예상 외의 오류가 발생할 수도 있다. 사용하지 않는다.
    • Camel case
      • 예: AreaOfTriangle, SearchArea, 등등
    • 밑줄(_; underscore)를 사용하는 방법
      • 예: area_of_triangle, search_area, 등등
  • 동적형태 (dynamic typing)
    • Python의 매우 중요한 개념
    • Python에서 사용하는 변수는 본래 형태를 갖고 있는 것이 아니라, 그 때 그 때 연결된 값의 형태를 대변해 줄뿐이라는 사실을 뜻한다.
      • >>>
      
>>>
      

      
>>>
      
>>>
      

      
>>>
      
>>>
      a = 234
      
print(type(a))
      
<class 'int'>
      
a = 2.34
      
print(type(a))
      
<class 'float'>
      
a = '2.34'
      
print(type(a))
      
<class 'str'>
      • 위 코드에서는 변수는 한 개만 사용한 반면 값은 세 개가 사용되고 있다.
      • 처음에는 정수에 변수가 연결되어 있기 때문에 변수의 형태를 물어본 것은 변수에 연결된 값, 이 경우 정수의 형태를 물어본 것이므로 결과는 정수라는 int가 출력되었으며
      • 그 다음으로 해당 변수를 다른 값, 이 경우에는 실수에 연결되었기 때문에 변수의 형태를 물어본 결과 실수 float임을 보여주고 있다.
      • 끝으로 해당변수를 문자열에 연결한 후의 변수의 형태를 물어본 경우에는 당연하게도 문자열을 뜻하는 str을 보여주고 있는 것이다.
    • Python의 이러한 성질을 글로 표현한 책이나 인터넷 정보는 쉽게 찾아볼 수 있다.
      • 하지만 그 내용의 몇 줄을 읽어 내려가다보면 변수에 값이 저장되어있다는 표현을 쉽게 사용하는 오류(?)를 발견할 수 있다.
      • Python의 변수에는 값이 저장되어 있는 것이 아니라 값과 연결되어 있을 뿐이다.
      • 하지만 사용하는 데 문제가 없다면 그냥 남들이 표현하듯이 저장되어 있다고 생각해도 좋다.
        • 이로 인해 밯생되는 오류에 대해서는 본인이 책임을 저야 한다.
        • 이와 관련된 사항으로 추후 shallow copy와 deep copy의 차이점을 살펴 볼 예정이다.
  • 할당문 (assignment statement)
    • 혹자는 대입문이라 부르기도 한다.
    • 변수에 값을 연결하는 가장 기본적인 구문이다.
    • 등호(=)를 사용하며 위 코드에서 보여준 것 처럼 등호 왼쪽에 적힌 이름의 변수에 연결한 값을 등호 오른쪽에 적는다.
      • 수학에서 배운 등호와 기호상으로는 동일하나 전혀 다르다는 사실을 빨리 인식해야 한다.
    • 때로는 변수에 연결할 값을 직접 적은 것 (literal)이 아니라 값을 얻는 표현식 (expression)을 적는 경우도 있다.
      • 또 다른 방법은 차차 알게 될 것이다.

    Python 변수 이름(names)과 값(literal values)

    • Python의 변수는 단순히 이름(name)뿐이며, 값(value)를 저장하는 대신 값에 연결되어있다.
      • Python이 아닌 다른 프로그래밍 언어, 예를 들어 c,에서는 변수에 값을 저장한다고 표현한다.
    • 동일한 값에 연결된 변수는 여러 개 있을 수 있으나, 여러 개 값에 연결된 변수는 존재하지 않는다.
      • 아래 코드의 두 변수는 동일한 값 5에 연결되어 있다.
      >>>
      a = b = 5
    • Python에서 변수에 값을 저장하느냐 하지 않느냐에 관해서는 나중에 리스트 등과 같이 여러 개 값을 묶은 데이터를 다룰 때 다시 한번 살펴볼 필요가 있다.
      • 종종 이와 관련하여 생각하지 못한 심각한 오류가 발생하기도 하며, deepcopy및 shallow copy등으로 표현하기도 한다.
    • 변수는 반드시 값에 연결되어야 한다.
      • 프로그램 입문자들이 초기에 많이 일으키는 오류는 바로 값에 연결되지 않은 변수를 사용하기 때문이다.
      • 아래 코드에서 오류가 발생 원인을 생각해 보라
        • >>>
        
>>>
        a = 5
        
print(a2+3)
        
Traceback (most recent call last):
        
  File "", line 1, in 
        
    print(a2+3)
        
NameError: name 'a2' is not defined. Did you mean: 'a'?
      • 오류 생성 원인은 변수 a2에 연결된 값과 5를 합한 값을 출력하라는 코드를 실행하는 단계에 있어서는 아직 이름이 a2라는 변수가 존재하지 않기 때문이다.
    • 경우에 따라 값이 아직 연결되지 않은 변수이름을 사용하는 경우도 있다.
      • 할당문(assignement statement)의 등호 왼쪽에 사용한 변수에는 기존에 값이 연결되어 있던 없더 상관하지 않고 무조건 등호 오른쪽에서 얻은 값을 연결한다.
      • 아래 코드와 같은 경우, 등호 오른쪽에 적힌 값 정수 5를 메모리에 저장한 후 저장주소를 변수 a에 연결해준다.
        • >>>
        a = 5
